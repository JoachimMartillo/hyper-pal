ADAM REST API Documentation

Resource: SavedView

About this resource

Representation of a saved view

Properties

Property Description Type
createdOn Gets the creation datetime in UTC time. datetime
dateAdded Gets the date when the saved view was shared if it was, or the date it was created on if it's not shared yet. This property will not be returned by default. In order to include the property in the response, add a header with the name 'select-savedview' and the value 'DateAdded' to your request. datetime
id Gets the Id of this saved view. guid
isPublic Gets a value indicating whether this saved view is public, meaning that a global shared link is available. boolean
isShared Gets a value indicating whether this saved view shared. This property will not be returned by default. In order to include the property in the response, add a header with the name 'select-savedview' and the value 'IsShared' to your request. boolean
lastActivityDate Gets the date this saved view was last used. This property will not be returned by default. In order to include the property in the response, add a header with the name 'select-savedview' and the value 'LastActivityDate' to your request. datetime
modifiedOn Gets the last modification datetime in UTC time. datetime
myLastActivityDate Gets the date this saved view was last used by the current user. This property will not be returned by default. In order to include the property in the response, add a header with the name 'select-savedview' and the value 'MyLastActivityDate' to your request. datetime
name Gets the name of the saved view. string
scope Gets the scope of this saved view. Scope is used to define logical groups of multiple view types. E.g. to group all views related to a specific site or application. string
tag Gets or sets the value for the Tag of this object. The value of this property should be valid Xml. This property will not be returned by default. In order to include the property in the response, add a header with the name 'select-savedview' and the value 'Tag' to your request. string
viewProperties Gets the data of this saved view. Dictionary of pairs with string keys and string values
viewType Gets the view type of this saved view. ViewType is used to store different view types used for the same scope. E.g. to differentiate between a search view and a collection view. string

Examples

Editing a saved view 
PUT /savedview/1fe6e87a31bc4208bd1ca4f000dbfaff
{
	"name": "mySavedView",
	"isPublic": false,
	"scope": "testScope",
	"viewType": "testViewType",
	"tag": "<xml>test tag</xml>",
	"viewProperties": {
		"myKey1": "myValue1",
		"myKey2": "myValue2"
	}
}

Note that when editing the propertyBag for a saved view, you always need to specify the entire propertyBag.
Deleting a saved view 
DELETE /savedview/8735a5a3e75a4bc0a2a1a3cd0069b669
Sharing a saved view
You can share a saved view via a PUT request to the share action endpoint: 
PUT /savedview/4535a5a3e75a4bc0a341a3cd0069b675/share
{
	"userIds": ["49631b3c-3a66-4c1f-8e21-a53e008e4716"],
	"userGroupIds": [],
	"updateRemoved":false
}
Parameter Description
userIds A list of Ids of users you want to share the view with.
userGroupIds A list of Ids of userGroups you want to share the view with.
updateRemoved A boolean value indicating whether or not you want to remove any shares that existed before but are not specified in the current request. Note that in order to remove all shares from a saved view, you must specify an empty array of userIds or userGroupIds in combination with the updateRemoved flag set to true. Passing null or not passing anything for these values is not allowed when clearing them.
Updating a saved view's last activity date
Updating a saved view's last activity date can be done by issuing a PUT request with an empty body to the following endpoint: 
PUT /savedview/4535a5a3e75a4bc0a341a3cd0069b675/lastactivitydate
The saved view's last activity date for the currently logged on user will be set to the current date and time.
