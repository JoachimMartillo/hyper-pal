Examples

Immediate propagation for classification updates.
For POST, PUT and DELETE operations on the /classification resource you can force an immediate update of the search index, or immediate propagation. To do so, add the boolean header/query-string parameter set-immediateSearchIndexUpdate and set it to "true". If you don't specify this parameter, or set it to "false", the change will be added to the queue of the Enterprise search synchronization process. Setting the parameter to true will immediately push the classification changes to the search index, so that this classification can immediately be found, e.g. while an ETL import is still running. This is important, for example, when importing both the child and parent classifications. 
PUT /classification/1fe6e87a31bc4208bd1ca4f000dbfaff
"set-immediateSearchIndexUpdate": "true",
{
"name": "Digital Images",
"identifer": "digital_images",
"sortIndex": 6,
...
}

Editing record or classification tree permissions on a classification

There are two different endpoints for classification permissions: recordpermissions and classificationtreepermissions

Both endpoints accept the same request body:

PUT /classification/8735a5a3e75a4bc0a2a1a3cd0069b669/{recordpermissions|classificationtreepermissions}
{
	"breakInheritance":false,
	"permissions": {
		"addOrUpdate": [
			{
					"userGroupId":"21d41eb183234b0cbe7b6a9c898b1de1",
					"accessRight":"Read"
			},
			{
					"userGroupId":"21d41eb183234b0cbe7b6a9c898b1de1",
					"accessRight":"FullControl"
			}
		],
		"remove": [
			{
					"userGroupId":"21d41eb183234b0cbe7b6a9c898b1de1
			}
		]
	}
}

Slave Classifications

A slave classification is a classification that is mapped to another classification, so that records automatically get linked to that slave classification once they are linked to the master classification. They can be configured through the creation of a classification and/or editing of a classification.

By adding the header value "select-classification: SlaveClassifications" all slave classifications of the current classification will be returned. 

GET /classification/id
select-classification: SlaveClassifications

The body of the editing request is in the following format:

PUT /classification/8735a5a3e75a4bc0a2a1a3cd0069b669
{
	"slaveclassifications": {
		"addOrUpdate": [ "ae82e88999d8479ab48ca3cd0069b4ba" ],
		"remove": [ "ae82e88999d8479ab48ca3cd0069b4ba" ]
	}
}
